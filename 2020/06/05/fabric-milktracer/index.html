fabric搭建一个牛奶溯源系统

发表于 2020-06-05 | 分类于 hyperledger > fabric | | 阅读次数: |
字数统计: 1k | 阅读时长 ≈ 3

食品溯源是区块链的一个不错的应用点，通过将食品的整个生产流程记录在链上，能够很好的治理食品安全问题。
这次更新一下fabric网络手动搭建的的第二部分，链码的安装调用
本文依旧主要参考《区块链开发实战》第12章内容
该博客所引用代码地址

系统架构

整过牛奶的生产过程被简化为三个步骤，奶牛场养殖，加工厂加工，销售终端出售。

所以我们的区块链网络架构就是由三个组织构成的。
分别是dairyfarm，machine和salesterminal。
另外每个组织要上链自己的信息，销售终端还应该可以查看牛奶的溯源信息。
所以应该有三个chaincode(智能合约)来实现各种功能。
另外还需要一个终端来输入各种调用命令。因为仅仅是输入fabric的命令，非常的繁琐。

网络搭建

前面已经介绍了整个系统的基本构成。为了模拟出这个环境，我们需要搭建一个有三个组织的区块链网络。并且每个组织分配至少一个节点。
如何搭建网络环境这里不做解释，请参考我的另一篇博客

链码的编写和安装

需要编写三个链码，origin_dairyfarm用来上链奶牛的一些信息；origin_machining用来上链工厂的加工分配信息；origin_salesterminal用来上链销售信息和查询溯源信息。
最终的目的就是对于特定的一瓶牛奶，能够查询出其从奶牛到奶瓶的整个过程。
书上的链码是用go语言写的，没有找到电子版，我码了一份，其中有一些名称的修改。代码地址请看开头链接。
链码编写好后需要打包并安装在三个组织peer节点，因为他们都是背书节点。
关于如何打包和安装请参考另一篇博客

nodejs客户端的编写

为了方便开发者编写fabric的应用，他们提供响应架构的sdk。本文采用nodejs搭建客户端。

下载fabric-sdk-node

去github上下载fabric-sdk-node到本地
在解压后的文件中执行

npm install

安装依赖。

nodejs客户端调用链吗的主要逻辑就是

1.启动服务等待命令
2.接收命令
3.加载帐号访问fabric接口
4.发送调用提案请求
5.发送调用求情
6.返回调用结果

其中第3部需要用的cryptogen生成的帐号。此外还可以通过部署fabric-ca-server服务器来生成帐号。

将编写好的fabricservice.js(接口的封装)和mainorigin.js(主逻辑)放入fabric-sdk-node文件夹中，然后执行:

http://localhost:3000/init

开启服务。

奶牛场上链操作举例:

http://localhost:3000/diaryfarm?parms=food //奶牛喂食

加工场上链操作举例:

http://localhost:3000/machining?parms=canned //机器灌装

销售终端上链操作举例：

http://localhost:3000/salesterminal?parms=distribution_time //机器灌装

溯源查询举例：

http://localhost:3000/getmilkhistory

问题与总结

在执行链码时，如果命令参数给的不够，会导致链码镜像崩溃关闭，查看日志原因是越界导致。所以应在编写链码时所处一定的判断，并给出一定的返回提示，防止链码镜像崩溃。
另外链码逻辑和客户端的编写为了做出简单的效果演示，逻辑十分不严密，很多地方是写死的，比如客户端默认就是查询cow_001（我改了下）的溯源信息，且上链时信息必须匹配，不然无法溯源。总之就是链码逻辑只考虑简单的演示，并没有任何容错可言。
